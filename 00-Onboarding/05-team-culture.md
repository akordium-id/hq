## üéØ Core Values

### **1. Pragmatic Innovation**
Kami percaya pada solusi yang praktis dan efektif, bukan teknologi untuk technologi itu sendiri.

**What this means:**
- Choose technology yang solves real problems
- Prefer simple solutions yang maintainable
- Innovation dengan purpose dan business value
- Consider long-term maintenance dan scalability

**In practice:**
- "Should we use Rust untuk project ini?" ‚Üí "Does Rust solve specific problems we have with Go?"
- "Let's use microservices" ‚Üí "What specific scaling problems are we solving?"

### **2. Documentation-Driven Excellence**
"Jika tidak tertulis di repository, maka itu tidak pernah terjadi."

**What this means:**
- Documentation adalah bagian dari pekerjaan, bukan tambahan
- Keputusan penting harus documented dan discoverable
- Single source of truth di GitHub
- Knowledge sharing adalah responsibility

**In practice:**
- Technical decisions di issue/discussion
- Architecture updates di repository documentation
- API documentation selalu up-to-date
- Meeting notes dan action items terdokumentasi

### **3. Async-First Collaboration**
Kami menghargai deep work dan komunikasi yang thoughtful.

**What this means:**
- Tidak ada expectation untuk immediate response
- Focus pada well-structured written communication
- Respect timezone dan deep work time
- Quality input over quick response

**In practice:**
- GitHub Issues untuk semua project communications
- Detailed PR descriptions dan review comments
- Weekly async updates daripada daily standups
- thoughtful responses daripada knee-jerk reactions

### **4. AI-Augmented Development**
Kami melihat AI sebagai partner, bukan replacement.

**What this means:**
- AI tools untuk productivity dan quality improvement
- Human creativity tetap utama
- Critical thinking dalam AI-generated code
- Continuous learning dan adaptation

**In practice:**
- Cursor IDE dengan AI pair programming
- GitHub Copilot untuk boilerplate dan suggestions
- AI untuk code review dan documentation
- Human verification untuk semua AI outputs

### **5. Craftsmanship Mindset**
Kami bangga dengan kualitas kerja yang kami hasilkan.

**What this means:**
- Code yang clean, maintainable, dan well-tested
- Attention to detail dalam UX dan implementation
- Pride dalam work yang kita lakukan
- Long-term thinking daripada quick fixes

**In practice:**
- Proper error handling dan edge cases
- Clean architecture dan separation of concerns
- Consistent coding standards
- Refactoring sebagai ongoing process

## üí¨ Communication Guidelines

### **Written Communication**
- **Clear dan concise:** Get to the point efficiently
- **Structured:** Use headings, lists, dan proper formatting
- **Actionable:** Include next steps dan owners
- **Contextual:** Provide enough background for understanding

### **Code Review**
- **Constructive:** Focus pada improvement, bukan criticism
- **Educational:** Explain "why" behind suggestions
- **Respectful:** Assume good intent
- **Thorough:** Cover security, performance, dan maintainability

### **Feedback Culture**
- **Regular:** Continuous feedback, bukan annual reviews
- **Two-way:** Everyone gives dan receives feedback
- **Specific:** Concrete examples dan actionable insights
- **Growth-oriented:** Focus pada development bukan judgment
